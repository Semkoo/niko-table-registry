## April 2026

### Faceted filters — correctness

- **`dynamicCounts` now honored in the auto-generated path of `useGeneratedOptionsForColumn`.** The generator already split row sources into `optionSourceRows` (driven by `limitToFilteredRows`) and `countSourceRows` (driven by `dynamicCounts`), but the auto-generated branch was building counts from `optionSourceRows` only — so setting `dynamicCounts: false` while keeping `limitToFilteredRows: true` silently had no effect on counts. Counts are now always computed from `countSourceRows`, consistent with the `augment` branch.
- **`dynamicCounts` now honored in the fallback path of `DataTableFacetedFilter` and `TableColumnFacetedFilterMenu`.** Previously, when a column didn't have `variant: "select" | "multi_select"` with `autoOptions` enabled, the fallback option generator ignored `dynamicCounts` entirely. The fallback now separates option discovery (from `optionRows`) from counting (from `countRows`), and the `useMemo` deps include `dynamicCounts` so React recomputes when it changes.
- **`TableColumnFacetedFilterMenu` now accepts `dynamicCounts` as a prop.** It was missing from the component's prop surface entirely; it is now a documented `boolean` prop, `@default true`, matching `DataTableFacetedFilter`.
- **`TableColumnFacetedFilterMenu` now forwards `dynamicCounts` to `useGeneratedOptionsForColumn`.** An earlier draft of this release added the prop and wired it into the fallback path, but the primary path (auto-generated options from `meta.variant: "select" | "multi_select"`) constructed the config as `{ limitToFilteredRows }` and silently dropped `dynamicCounts`, so the base hook always saw the default `true`. The prop now flows through both paths in `TableColumnFacetedFilterMenu`, matching the parallel wiring already present in `DataTableFacetedFilter`.

### Faceted filters — UX default

- **Multi-select filters default `limitToFilteredRows` to `false`.** Single-select keeps the previous `true` default. Rationale: a multi-select faceted filter that narrows its own option list as rows filter out causes a classic broken-facet experience — unchecking the last selected value can also remove _other_ still-selected values from the visible list, because their rows have just been filtered away. Multi-select filters now show the full option universe by default; single-select continues to narrow as the user refines. The new default is expressed as `limitToFilteredRows ??= !multiple`, so explicit `limitToFilteredRows` props always win.
- **`DataTableFacetedFilter` prop docs** now describe the default as `!multiple` with both cases explained.

### Faceted filters — options with zero counts

- **Options discovered from `optionRows` are always returned**, even when their count in `countRows` is `0`. Previously, options with count `0` were silently dropped because the iteration used a single `valueCounts` map that conflated "seen" with "counted." This only surfaces when `limitToFilteredRows !== dynamicCounts`, but when it does it produced confusing "disappearing option" behavior.

### Tests — first behavioral test suite for the library

- **Introduced [Vitest](https://vitest.dev) as the unit/component test runner.** Added `vitest`, `@vitejs/plugin-react`, `@testing-library/react`, `@testing-library/dom`, `jsdom`, and `@vitest/ui` as dev dependencies. New scripts: `npm run test` (run once) and `npm run test:watch` (watch mode). Tests live next to the code they cover, in `src/**/*.{test,spec}.{ts,tsx}`.
- **`useGeneratedOptions` — 13 behavioral tests** ([`use-generated-options.test.tsx`](https://github.com/Semkoo/niko-table-registry/tree/main/src/components/niko-table/hooks/use-generated-options.test.tsx)) covering every relevant `(limitToFilteredRows, dynamicCounts, mergeStrategy)` permutation against a real TanStack table: the `limitToFilteredRows=true, dynamicCounts=false` case that this release actually fixes, the zero-count guard for `limitToFilteredRows=false, dynamicCounts=true`, and the `preserve` / `augment` merge-strategy branches. Tests use explicit `filterFn`s on the fixture columns so row filtering actually happens — a subtlety worth noting for future contributors.
- **`TableColumnFacetedFilterMenu` — 6 wiring tests** ([`table-column-faceted-filter.test.tsx`](https://github.com/Semkoo/niko-table-registry/tree/main/src/components/niko-table/filters/table-column-faceted-filter.test.tsx)) pin down the exact bug that shipped silently in an earlier draft of this release: the component must forward `dynamicCounts` to `useGeneratedOptionsForColumn`, and it must apply `limitToFilteredRows ??= !multiple` before calling the hook. The tests use `vi.mock` to intercept the hook call and assert on the config object directly.
- **`DataTableFacetedFilter` — 4 wiring tests** ([`data-table-faceted-filter.test.tsx`](https://github.com/Semkoo/niko-table-registry/tree/main/src/components/niko-table/components/data-table-faceted-filter.test.tsx)) mirror the direct-prop tests for the context-aware entry point. Mocks `useDataTable` to inject a real TanStack table and verifies that `dynamicCounts` forwarding and the `!multiple` default both resolve correctly before the hook call.
- **Registry example updated to exercise the new default.** [`faceted.tsx`](https://github.com/Semkoo/niko-table-registry/tree/main/src/registry/new-york/examples/niko-table/faceted.tsx) previously set `limitToFilteredRows={false}` explicitly on its multi-select category filter. The explicit prop was removed so that the example now relies on — and visibly demonstrates — the new `!multiple` default. Any regression that breaks the default behavior will show up in the docs site.

### Documentation

- **`DataTableFacetedFilter` — `limitToFilteredRows` row in props table** now shows `!multiple` as the default with both branches described.

### Internals — notes on the refactor

- `useFacetedOptions` in `data-table-faceted-filter.tsx` and `TableColumnFacetedFilterMenu`'s fallback now walk rows in two passes when `optionRows !== countRows`: the first pass builds the available-option set from `optionRows`, the second counts occurrences from `countRows`. When both flags agree (the common case, and the default), you still get two passes — optimizing this to a single pass via reference-equality fast-path is a planned follow-up (see [use-generated-options.ts](/niko-table/overview/hooks/)).
- `useFacetedOptions` now applies `limitToFilteredRows = true` as a destructuring default, matching the base `useGeneratedOptions` hook. Before this, the parameter had no default — so if a future caller forgot to resolve `!multiple` before calling, `undefined` would be treated as `false` in the fallback and explicit-options paths but as `true` (via the base hook's own default) in the auto-generated path, producing inconsistent behavior across branches. Current callers (`DataTableFacetedFilter`, `DataTableFacetedFilterContent`) resolve `limitToFilteredRows ??= !multiple` at the component entry point before invoking the hook, so explicit `limitToFilteredRows` props and the `!multiple` default continue to win over the hook-level fallback.
- The `preserve` merge strategy in `useGeneratedOptionsForColumn` is unchanged in behavior: it continues to return user-defined `meta.options` **untouched** (no injected counts), respecting `limitToFilteredRows` only to hide values that aren't present in the current option-source rows. Counts are still only applied under `mergeStrategy: "augment"`. An earlier draft of the refactor injected counts into preserved options — that draft was corrected before this release.

## March 2026

### Performance & consistency

- **Row click: event delegation** — All table body components (standard, virtualized, and DnD variants) now use a single delegated row-click handler instead of one handler per row or per cell. This reduces allocations and keeps behavior consistent across `DataTableBody`, `DataTableVirtualizedBody`, `DataTableVirtualizedDndBody`, `DataTableVirtualizedDndColumnBody`, `DataTableDndBody`, and `DataTableDndColumnBody`.
- **DataTableRoot** — Global filter and default column-pinning handlers are memoized with `useCallback` so table options stay stable and the table does not re-initialize unnecessarily.
- **TablePagination** — Page size, page input, and prev/next button handlers are memoized with `useCallback` for stable references when pagination state changes.
- **DataTableAside** — Trigger toggle and close button handlers are memoized with `useCallback`.

### Documentation

- **DataTableRoot** — Docs now list the required `children` prop and the optional `state` prop for controlled mode. The introduction page also documents `className`.
- Props tables and examples were reviewed for accuracy against the current implementation.

### Bug Fixes

- **TablePagination — page input draft state** — The page number input now uses local draft state so users can type intermediate values (e.g. `"1"` before completing `"12"`) without the table jumping mid-edit. The page index is only committed on blur or Enter; invalid input resets to the current page.
- **TablePagination — stale page index on page size change** — `onPageSizeChange` now receives the recalculated page index (`Math.floor((pageIndex * pageSize) / newPageSize)`) instead of the stale pre-change index.
- **Sticky header z-index** — Fixed an issue where pinned body cells could overlap the sticky header during scroll. The sticky header container now uses `z-30`, above header pinned cells (`z-20`) and body pinned cells (`z-10`).
- **Pinned header cell `top` style** — Added `top: 0` to pinned header cells in `getCommonPinningStyles` so they stick vertically when the header is sticky.

## February 2026 - Initial Release

The first public release of Niko Table — a composable, shadcn-compatible data table component registry built with TanStack Table and React.

### Components

- **DataTable** - Core data table with sorting, filtering, and pagination
- **DataTableVirtualized** - Virtualized rendering for large datasets
- **DataTablePagination** - Flexible pagination controls
- **DataTableSearchFilter** - Global search across all columns
- **DataTableFacetedFilter** - Multi-select faceted filtering with counts
- **DataTableFilterMenu** - Advanced filter menu with AND/OR logic
- **DataTableInlineFilter** - Inline filter bar with mixed operator support
- **DataTableSliderFilter** - Range slider filtering for numeric columns
- **DataTableDateFilter** - Date and date range filtering
- **DataTableExportButton** - CSV/JSON export
- **DataTableAside** - Side panel for row details
- **DataTableSelectionBar** - Bulk actions on selected rows
- **DataTableColumnSort** - Sort menu and sort icon components
- **DataTableColumnHide** - Column visibility toggle
- **DataTableColumnPin** - Column pinning (left/right)

### Features

- **Row DnD** - Drag and drop row reordering via `@dnd-kit`
- **Column DnD** - Drag and drop column reordering
- **Row Selection** - Single and multi-row selection with bulk actions
- **Row Expansion** - Expandable rows with custom content
- **Tree Table** - Hierarchical data with expand/collapse
- **Virtualization** - Smooth scrolling for 10k+ rows
- **URL State** - Sync filters, sorting, and pagination to URL via `nuqs`
- **Server-Side** - Server-side filtering, sorting, and pagination support

### Filter System

- 15+ filter operators (contains, equals, greater than, between, etc.)
- Mixed AND/OR logic with mathematical precedence
- Per-filter join operator control
- Regex-cached filter execution for large datasets
