Follow this when adding or restructuring docs. The site is Astro Starlight; demos come from registry example files under `src/registry/new-york/examples/`.

## Information architecture

Split **API / overview** from **runnable examples**. Do not put the same topic in both sidebars.

| Section                | Purpose                                                | Path                           |
| ---------------------- | ------------------------------------------------------ | ------------------------------ |
| **Niko Table**         | Introduction + overview by layer (Components, Core, ‚Ä¶) | `src/content/docs/niko-table/` |
| **Examples**           | Table demos (Simple Table, Faceted Filter, ‚Ä¶)          | `src/content/docs/examples/`   |
| **Data Grid**          | Introduction + overview by layer + API Reference       | `src/content/docs/data-grid/`  |
| **Data Grid Examples** | Grid demos (Basic Grid, Cell Types, Validation, ‚Ä¶)     | `src/content/docs/examples/`   |

**Data Grid** overview pages mirror the source tree (`grid/components`, `grid/core`, `grid/hooks`, ‚Ä¶). Feature walkthroughs with a live demo belong under **Data Grid Examples**, not under overview.

Reference: [Basic Grid](/examples/basic-grid/) ‚Äî minimal grid example that follows this guide.

- Table example slugs: `examples/<feature>-table` (or existing names like `simple-table`).
- Grid example slugs: `examples/<feature>-grid`.
- Overview never duplicates an example‚Äôs full tutorial; link across instead.

## Adding a Data Grid Example

<Steps>

1. **Registry demo** ‚Äî add a self-contained example under `src/registry/new-york/examples/niko-table/` (e.g. `grid-foo.tsx`). Import only from `@/components/niko-table/...` direct paths and `@/components/ui`. Use deterministic mock data (no `Math.random()` / `Date.now()`).

2. **Register the demo name** ‚Äî add the string to `CodePreviewInternal`‚Äôs `Demo` union in `src/components/markdown/code-preview/_code-preview-internal.tsx`, and wire the lazy import map there.

3. **Example MDX** ‚Äî create `src/content/docs/examples/<name>-grid.mdx` with:

   - Short pitch (1‚Äì2 sentences)
   - `<CodePreview demo="niko-table/‚Ä¶" />`
   - Optional `<CollapsiblePreview>` + state demo
   - Installation (`CliCommandCode`)
   - Optional short ‚ÄúWhat‚Äôs included‚Äù / usage notes
   - Next links to related overview or examples

4. **Sidebar** ‚Äî add an entry under **Data Grid Examples** in `astro.config.mts` (`slug: "examples/<name>-grid"`).

5. **Cross-links** ‚Äî from Introduction / overview pages, link to the example; don‚Äôt paste a second live demo of the same feature into overview.

</Steps>

## Page patterns

### Overview (API by layer)

- Source Code / related DocsLink row at the top when useful
- Props tables and import paths by file
- Point to examples for ‚Äúsee it running‚Äù

### Example (demo-first)

- Demo first, prose after
- Installation for that example‚Äôs registry items only
- Keep deeper theory in overview; keep the example focused

### Links

Prefer trailing slashes in hardcoded `href`s and markdown links, e.g. `/examples/basic-grid/`. Dev can accept both with `trailingSlash: "ignore"`, but slashed links stay consistent with production directory URLs.

## Sidebar scroll

Theme-black does not use Starlight‚Äôs `#starlight__sidebar` / `SidebarPersister`. **Do not** add that id ‚Äî it freezes the main thread with this theme.

Scroll restore for the theme sidebar lives in `src/components/overrides/head.astro` (`aside.aside .sidebar` + `sessionStorage`). Prefer extending that script over a custom Sidebar override.

## Checklist before PR

- [ ] Overview vs Examples split is clear (no duplicate nav items)
- [ ] Demo is registered and renders in `CodePreview`
- [ ] Mock data is deterministic
- [ ] Sidebar entry added in `astro.config.mts`
- [ ] Internal links point at the new paths
- [ ] `pnpm run build` (or at least `astro check`) passes
